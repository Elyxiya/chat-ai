## 关键 Mock 策略

| 场景 | 策略 |
|------|------|
| Zustand Store | `vi.mock('@/api/client')` Mock API 模块，直接调用 store actions 验证 state |
| WebSocket | `vi.mock('socket.io-client')` + `vi.hoisted` mockSocket，手动触发事件 handlers |
| SSE 流式 | `createMockReader` 模拟 ReadableStream，构造 SSE 格式 `data: {...}\n\n` |
| 路由导航 | Mock `useNavigate`，使用 `MemoryRouter` 包裹组件 |
| localStorage | `vi.hoisted` + `Object.defineProperty` 在模块加载前替换 |
| React 组件 | React Testing Library render + fireEvent + `@testing-library/jest-dom` 断言 |
| Fetch API | `globalThis.fetch = vi.fn().mockResolvedValue(...)` 全局替换 |

pnpm --filter ai-native-chat-web test:watch

pnpm --filter ai-native-chat-web test:coverage

# 运行特定测试文件
pnpm --filter ai-native-chat-web exec vitest run stores/auth.store.spec.ts
