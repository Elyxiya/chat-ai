## 关键修复记录

### Redis Mock (ioredis ES Module)
`ioredis` 使用 ES module `default` 导出，Jest (CommonJS) 中必须使用 `{ default: fn, Redis: fn }` 格式才能正确模拟。

### Planning Engine Mock 格式
| 问题 | 修复 |
|------|------|
| LLM 返回 `{ choices: [...] }` 对象 | 改为返回纯字符串（服务层直接使用字符串响应） |
| 工具调用格式 `行动: get_time` | 改为 `行动: get_time()`（正则要求括号） |
| 流式 yield 纯字符串 | 改为 yield `{ type: 'chunk', data: '...' }` 事件对象 |
| reflect 响应为纯文本 | 改为 JSON 格式 `{"needsRetry":false,"summary":"..."}` |
| 流式生成器非 async iterable | 使用 `mockImplementation(async function* () { ... })` |

### Chat Controller markRead 签名
`markRead(userId, sessionId)` → `markRead(userId, { sessionId, lastMessageId })` (DTO 对象传参)

## 执行方式

```bash
# 运行所有测试
pnpm --filter @app/api test

# 运行特定文件
pnpm --filter @app/api test -- --testPathPattern="chat.service"

# 带覆盖率报告
pnpm --filter @app/api test -- --coverage

# 生成 HTML 覆盖率报告
pnpm --filter @app/api test -- --coverage --coverageReporters=html
```

HTML 覆盖率报告位于 `apps/api/coverage/lcov-report/index.html`，可在浏览器中查看详细的逐文件、逐行覆盖率。
