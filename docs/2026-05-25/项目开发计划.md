## 计划总览

本次开发按优先级分为 3 个 Sprint，覆盖质量基建、功能完善、体验扩展三个阶段。

---

## Sprint 1：质量加固与基建

### 1.1 CI/CD 流水线
- **文件：** `.github/workflows/ci.yml`
- **内容：** pnpm install → lint → typecheck → test (API+Web) → build
- **触发：** push 到 main / pull request

### 1.2 前端覆盖率提升至 60%+
- **目标文件：** AgentChatPage, KnowledgePage, SettingsPage, GroupManagementPage, knowledge.store, notification.store, theme.store, FileUploadPanel, NotificationPanel, UserSearchModal, ToolCallLog
- **框架：** Vitest + React Testing Library
- **达标：** 各新增测试文件 Stmts ≥ 80%

### 1.3 后端覆盖率提升至 80%+
- **目标文件：** chat.gateway (68%→80%), knowledge.controller, agent.controller, upload.controller, upload.service, minio.service
- **框架：** Jest + @nestjs/testing
- **达标：** 各新增测试文件 Stmts ≥ 80%

### 1.4 E2E 测试（Playwright）
- **文件：** `apps/api/test/e2e/` + playwright.config
- **覆盖流程：** 注册登录 → 创建私聊 → 发送消息 → AI 对话 → 知识库操作
- **工具：** Playwright + 自建测试用户

---

## Sprint 2：功能完善

### 2.1 文件预览与下载
- **后端：** `GET /upload/files/:id/download` — 文件下载 endpoint（流式返回 + Content-Disposition）
- **前端：** FilePreviewModal 组件 — 图片内联预览、其他文件信息展示+下载按钮
- **消息：** 消息气泡中图片消息可点击放大

### 2.2 消息搜索功能
- **后端：** `GET /chat/sessions/:id/search?q=` — Prisma 全文搜索 session 内消息
- **前端：** 搜索输入栏（会话列表顶部），搜索结果高亮显示

### 2.3 用户在线状态实时同步
- **后端：** ChatGateway 已实现 `online_users` / `user_status` 事件
- **前端：** chat.store 处理在线状态事件，SessionList 显示状态指示器（绿点/灰点）

### 2.4 EnhancedAgentPage 集成
- 将现有 EnhancedAgentPage 接入路由 `chat/agent`，替换原 AgentChatPage
- 支持三种模式：ReAct / Planner / Reasoner
- 完整显示 StreamingText + ThinkingChain + ToolCallLog

### 2.5 通知系统前端对接
- NotificationPanel 对接后端真实通知数据
- 支持已读/未读切换、批量已读、点击跳转对应会话

---

## Sprint 3：体验与扩展

### 3.1 移动端响应式适配
- `<md` 断点：侧边栏折叠为抽屉式，输入区自适应，消息气泡宽度 85%
- `<768px` 和 `<1024px` 断点优化

### 3.2 Admin 后台管理模块
- **后端：** AdminModule — 用户列表、禁言/封禁、系统配置、审计日志查看
- **前端：** AdminPage — 用户管理表格、系统设置表单

### 3.3 性能优化
- 消息列表虚拟滚动（react-window）
- 图片懒加载（IntersectionObserver）

### 3.4 深色模式细节打磨
- 修复已知对比度问题
- 添加主题过渡动画
- 三档主题切换无闪烁
